# 智能视频流分析平台 — 项目结构说明

> **项目名称**: Stream Analyzer (IVSAP - Intelligent Video Stream Analysis Platform)  
> **技术栈**: 前端 Vite + 原生 JS，后端 Node.js + Express + PostgreSQL

---

## 一、项目概览

本项目是一个智能视频流分析平台，提供视频推流管理、AI 检测告警、工单分发、人员权限管理等能力。采用前后端分离架构：

- **前端**：纯原生 JS（无框架）+ Vite 构建，浏览器端通过 `window.DB` 对象管理数据
- **后端**：Node.js + Express + PostgreSQL，提供 RESTful API + WebSocket
- **部署**：支持 Docker Compose 一键部署

前端可独立运行（使用 `src/db.js` 中内置的模拟数据），也可连接后端 API 获取真实数据。

## 三、前端核心文件详解

### 1. `index.html` — 应用入口

SPA 单页应用模板，包含所有页面的公共结构：

| 区域 | ID | 说明 |
|------|------|------|
| 侧边导航 | `#sidebar` | 8 个导航项：工作台、推流管理、检测项目、算法管理、告警中心、工单分发、人员权限、系统管理 |
| 主内容区 | `#mainContent` | 顶部栏（标题+时钟+通知）+ 页面容器 `#pageContainer` |
| 弹窗 | `#modalOverlay` | 模态对话框，用于表单/详情 |
| Toast | `#toastContainer` | 通知消息容器 |

**脚本加载顺序**（严格按此顺序）：

```html
<script src="src/db.js"></script>   <!-- 1. 初始化数据层 -->
<script src="src/api.js"></script>  <!-- 2. 初始化 API 客户端 -->
<script src="src/main.js"></script> <!-- 3. 启动应用 -->
```

---

### 2. `src/style.css` — 全局样式表

约 416 行，政务风格设计：

- **字体**：宋体系列（SimSun, STSong, Songti SC），正式稳重
- **配色**：蓝色为主色（`--primary: #1a56db`），黑白灰色系辅助
- **布局**：左侧固定 240px 侧边栏 + 右侧弹性主内容区
- **组件样式**：卡片、表格、表单、标签、看板、弹窗、日历、通知流、组织树等
- **响应式**：768px 以下侧边栏变为隐藏式抽屉导航

---

### 3. `src/db.js` — 前端数据层（模拟数据）

**约 10KB**，定义 `window.DB` 全局对象，包含完整的模拟数据集：

| 数据字段 | 类型 | 说明 | 示例数据量 |
|---------|------|------|-----------|
| `streams` | Array | 视频流列表 | 6 条 |
| `detections` | Array | 检测项目 | 4 条 |
| `alerts` | Array | 告警记录 | 8 条 |
| `tickets` | Array | 工单记录 | 5 条 |
| `users` | Array | 人员信息 | 6 人 |
| `roles` | Array | 角色定义 | 4 种 |
| `depts` | Array | 部门树 | 4 个部门 |
| `algoTypes` | Array | 算法类型 | 4 种（含 AI 提示词） |
| `apiModels` | Array | 大模型配置 | 3 个（GPT-4o、通义千问、DeepSeek） |
| `schedules` | Array | 排班数据 | 5 条 |
| `ticketLogs` | Array | 操作日志 | 5 条 |
| `config` | Object | 系统配置 | 4 项 |

**特性**：

- 自执行 IIFE，从 `localStorage` 恢复数据（`key: ivsap_db_data`）
- 暴露 `window.DB._save()` 方法保存到 localStorage
- 页面关闭自动保存 + 每 15 秒自动保存
- **在不启动后端时，应用依靠本文件的数据独立运行**

---

### 4. `src/api.js` — API 客户端

**约 8KB**，前后端通信层：

**核心函数**：

| 函数 | 说明 |
|------|------|
| `request(method, path, body)` | 通用 HTTP 请求，自动带上 JWT Token |
| `showLoginPage()` | 渲染登录表单（替换 `#app` 内容） |
| `doLogin()` | 调用 `/api/auth/login`，保存 token，加载数据 |
| `loadAllData()` | 并行加载所有后端数据（13 个 API 请求），填充到 `window.DB` |
| `buildDeptTree()` | 将扁平部门列表构建为树形结构 |

**自动初始化逻辑**：

```
页面加载 → api.js autoInit()
  ├─ 有 token → loadAllData() → APP_INIT()
  │             └─ 失败 → 降级使用 db.js 模拟数据
  └─ 无 token + 无 db.js 数据 → 显示登录页
     └─ 无 token + 有 db.js 数据 → 由 main.js 直接启动
```

**后端 API 保存代理**：`window.API_SAVE` 对象提供了所有后端写操作的接口（创建推流、更新告警、分配工单等）。

---

### 5. `src/main.js` — 主应用逻辑

**约 87KB**，**整个前端的核心**，包含约 2000 行代码：

#### 页面渲染函数

| 函数 | 页面 | 说明 |
|------|------|------|
| `renderDashboard()` | 工作台 | 统计卡片、快捷操作、视频流卡片、实时告警流、趋势图表占位 |
| `renderStreams()` | 推流管理 | 批量操作栏 + 视频流表格（含选择框、ping 检测） |
| `renderDetection()` | 检测项目 | 检测项目表格 + ROI 配置面板 + 灵敏度滑块 |
| `renderAlgorithms()` | 算法管理 | 算法类型卡片列表（含提示词编辑弹窗） |
| `renderAlerts()` | 告警中心 | 过滤栏 + 告警表格 + 类型统计卡片 |
| `renderTickets()` | 工单分发 | 列表/看板两种视图切换 |
| `renderUsers()` | 人员权限 | 人员管理 / 角色权限 / 排班日历 三个 Tab |
| `renderSystem()` | 系统管理 | 大模型配置 + 系统设置 + 操作日志 |

#### 辅助函数

| 函数 | 说明 |
|------|------|
| `html()` | 模板字面量辅助，简化 HTML 拼接 |
| `toast()` | 通知消息显示（自动消失） |
| `modal()` | 弹窗对话框（支持表单、详情等） |
| `navigate()` | 页面路由切换（更新导航高亮 + 渲染对应页面） |
| `simulateAlert()` | 模拟实时告警（每隔一段时间自动生成新告警） |
| `applyRolePermissions()` | 根据角色权限控制导航菜单显示 |
| `updateNavBadges()` | 更新告警/工单导航徽标数 |

#### 数据 CRUD 函数

包含完整的 CRUD 操作（约 50+ 个函数）：

- **推流**：`showAddStream()`、`saveStream()`、`editStream()`、`deleteStream()`、`batchStreamAction()`
- **检测**：`showAddDetection()`、`toggleDetection()`、`editDetection()`
- **告警**：`handleAlert()`、`batchCreateTicket()`
- **工单**：`showCreateTicket()`、`assignTicket()`、`approveTicket()`、`rejectTicket()`
- **人员**：`showAddUser()`、`saveEditUser()`、`toggleUser()`
- **角色**：`showAddRole()`、`saveEditRole()`
- **系统**：`showEditApiModel()`、`saveSystemConfig()`、`saveTicketRules()`

#### 初始化流程

```
DOMContentLoaded
  └─ window.DB 就绪？
       ├─ 是 → APP_INIT()
       │        ├─ 绑定导航点击事件
       │        ├─ 绑定通知按钮事件
       │        ├─ 启动时钟（每秒更新）
       │        ├─ 应用角色权限
       │        ├─ 更新导航徽标
       │        ├─ 启动数据清理（每 24h）
       │        └─ navigate('dashboard') 默认显示工作台
       └─ 否 → 轮询等待（最多 10 秒超时）
```

## 四、后端核心文件详解

### 1. `server/index.js` — 后端入口

- Express 服务器，端口 **3001**
- 注册所有路由（`/api/auth`、`/api/streams` 等）
- CORS 跨域配置
- WebSocket 实时推送（`/ws`）

### 2. 数据库 — `server/db/`

| 文件 | 说明 |
|------|------|
| `pool.js` | PostgreSQL 连接池，基于 `pg` 库 |
| `migrate.js` | 执行 `migrations/001_init.sql` 建表 |
| `seed.js` | 插入演示种子数据 |
| `001_init.sql` | 初始建表：users, streams, detections, alerts, tickets, roles, depts, api_models, schedules, ticket_logs, config |

### 3. API 路由 — `server/routes/`

| 路由文件 | 端点前缀 | 主要功能 |
|---------|---------|---------|
| `auth.js` | `/api/auth` | 登录（JWT 签发） |
| `streams.js` | `/api/streams` | 视频流 CRUD + 批量操作 |
| `detections.js` | `/api/detections` | 检测项目 CRUD + 启动/停止 |
| `alerts.js` | `/api/alerts` | 告警查询 + 状态变更 |
| `tickets.js` | `/api/tickets` | 工单 CRUD + 分配/处置/验收/驳回 |
| `algorithms.js` | `/api/algorithms` | 算法类型管理 |
| `users.js` | `/api/users` | 人员 CRUD + 启用/禁用 |
| `depts.js` | `/api/depts` | 部门管理 |
| `roles.js` | `/api/roles` | 角色 CRUD |
| `system.js` | `/api/system` | 系统设置、大模型配置、操作日志、排班 |
| `ws.js` | `/ws` | WebSocket 实时推送告警 |

### 4. `server/middleware/auth.js` — JWT 中间件

验证请求头中的 `Bearer token`，保护所有业务接口。

## 五、部署相关文件

### 1. `vite.config.js` — 构建配置

- 开发服务器端口 `3000`
- API 代理：`/api` → `localhost:3001`，`/ws` → `ws://localhost:3001`
- 自定义插件 `serveRawJs`：开发环境下，`main.js`/`db.js`/`api.js` 跳过 esbuild 转换（因为代码中使用模板字符串拼接 HTML，与 JSX 语法冲突）

### 2. `docker-compose.yml` — Docker 编排

```yaml
服务:
  postgres:  PostgreSQL 16 数据库
  migrate:   执行数据库迁移（依赖 postgres 就绪）
  seed:      填充种子数据（依赖 migrate 完成）
  api:       后端 API 服务（端口 3001，依赖 seed 完成）
```

### 3. `package.json` — 脚本使用

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动前端开发服务器（端口 3000） |
| `npm run dev:server` | 启动后端服务器（端口 3001） |
| `npm run build` | 构建前端到 `dist/` 目录 |
| `npm run setup` | 一键初始化后端（npm install + migrate + seed） |

**两种运行模式**：

1. **独立模式**（无后端）：仅启动 Vite 开发服务器，`db.js` 提供模拟数据，`api.js` 检测无 token 且无后端时，`main.js` 直接初始化应用
2. **完整模式**（有后端）：先启动后端（`docker-compose up`），再启动前端，`api.js` 从后端 API 加载真实数据
